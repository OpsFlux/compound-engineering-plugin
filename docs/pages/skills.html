技能参考

将技能视为克劳德·科德可以在对话中阅读的参考手册。当您编写 Rails 代码并想要 DHH 的风格，或者像 Andrew Kane 那样构建 gem 时，您不需要粘贴文档 - 只需调用该技能即可。克劳德阅读它，吸收其中的模式，并以这种方式编写代码。

如何使用技能

# In your prompt, reference the skill
skill: [skill-name]

# Examples
skill: gemini-imagegen
skill: dhh-ruby-style
skill: create-agent-skills

技能与特​​工

代理商 是角色——他们 do 事物。 技能 是知识——他们教克劳德 如何 做事。使用 claude agent [name] 当您希望有人检查您的代码时。使用 skill: [name] 当您想自己编写特定风格的代码时。

开发工具 (7)

这些技能教会了克劳德特定的编码风格和架构模式。当您想要遵循特定理念的代码时使用它们——不仅仅是任何工作代码，而是看起来像是由特定人员或框架编写的代码。

dhh 红宝石风格

导轨

您需要五行而不是五十行的 Rails 控制器。处理授权、广播和业务逻辑的模型，无需到处都有服务对象。这项技能教会 Claude 以 DHH 在 37signals 编写代码的方式编写代码 — REST 纯、Hotwire 优先、无建筑航天学。

关键模式

  • 休息纯度 - 仅 7 个 REST 动作
  • 脂肪模型 - 模型中的业务逻辑、授权、广播
  • 瘦控制器 - 1-5行动作
  • 当前属性 - 请求上下文
  • 热线/涡轮 - 模型级广播

Ruby 语法首选项

  • 符号数组 %i[...]
  • 现代哈希语法
  • 简单条件的三元数
  • 突变的 Bang 方法
skill: dhh-ruby-style

安德鲁·凯恩宝石作家

红宝石

Andrew Kane 编写了 100 多篇 Ruby gem，下载量达到 3.74 亿次。每个 gem 都遵循相同的模式：最小依赖、类宏 DSL、无 Rails 耦合的 Rails 集成。当您构建 gem 并希望它从第一天起就可以投入生产时，您可以这样做。

哲学

  • 简单胜过聪明
  • 零或最小依赖性
  • 关于元编程的显式代码
  • 无需 Rails 耦合的 Rails 集成

关键模式

  • 用于配置的类宏 DSL
  • ActiveSupport.on_load 用于 Rails 集成
  • class << self with attr_accessor
  • 挂钩的铁轨图案
  • Minitest（无 RSpec）

参考文件

  • references/module-organization.md
  • references/rails-integration.md
  • references/database-adapters.md
  • references/testing-patterns.md

前端设计

设计

您已经看到了 AI 通常生成的内容：Inter 字体、紫色渐变、所有内容的圆角。这项技能教会克劳德设计出与其他人工智能生成的网站不同的界面。它涉及有目的的排版、意想不到的调色板和个性化的界面。

设计思维

  • 目的 - 接口有什么用？
  • 语气 - 它应该唤起什么样的感觉？
  • 约束条件 - 技术和品牌限制
  • 差异化 - 如何脱颖而出

重点领域

  • 具有独特字体选择的版式
  • 颜色和主题与 CSS 变量的一致性
  • 运动和动画模式
  • 不对称的空间构图
  • 背景（渐变、纹理、图案）

避免通用的 AI 美学，例如 Inter 字体、紫色渐变和圆角。

复合文档

文档

经过一个小时的调试后，您刚刚修复了一个奇怪的构建错误。明天你就会忘记你是如何修复它的。此技能会自动检测您何时解决某些问题（诸如“有效”或“已修复”之类的短语），并使用 YAML frontmatter 对其进行记录，以便您可以再次找到它。每个记录的解决方案都会丰富您团队的知识。

自动触发

短语：“有效”、“已修复”、“现在有效”、“问题已解决”

7 步流程

  1. 检测确认短语
  2. 收集背景信息（模块、症状、调查、根本原因）
  3. 检查现有文档是否存在类似问题
  4. 生成文件名
  5. 验证 YAML frontmatter
  6. 在类别目录中创​​建文档
  7. 交叉引用相关问题

类别

  • build-errors/
  • test-failures/
  • runtime-errors/
  • performance-issues/
  • database-issues/
  • security-issues/

内容和工作流程 (3)

写作、编辑和组织工作。这些技能处理从风格指南合规性到 git 工作树管理的一切事务——元工作使实际工作变得更容易。

每种风格的编辑器

内容

您写了一份草稿，但不确定它是否符合 Every 的风格指南。 “internet”应该大写吗？允许这种逗号拼接吗？该技能进行四个阶段的逐行审查：上下文、详细编辑、机械检查和可行的建议。这就像拥有一个永不疲倦的文案编辑。

四阶段审查

  1. 初步评估 - 语境、类型、受众、语气
  2. 详细线路编辑 - 句子结构、标点符号、大小写
  3. 机械评论 - 间距、格式、一致性
  4. 建议 - 可操作的改进建议

风格格纹

  • 语法和标点符号
  • 风格指南合规性
  • 大写规则
  • 选词优化
  • 格式一致性
skill: every-style-editor

文件全部

工作流程

你的待办事项列表是一堆 markdown 文件 todos/ 目录。每个文件名都编码状态、优先级和描述。没有数据库，没有 UI，只有带有 YAML frontmatter 的文件。当您需要在不设置 Jira 的情况下跟踪工作时，这就是系统。

文件格式

# Naming convention
{issue_id}-{status}-{priority}-{description}.md

# Examples
001-pending-p1-security-vulnerability.md
002-ready-p2-performance-optimization.md
003-complete-p3-code-cleanup.md

状态值

  • pending - 需要分诊
  • ready - 批准工作
  • complete - 完毕

优先值

  • p1 - 批判的
  • p2 - 重要的
  • p3 - 值得拥有

YAML 前沿问题

---
status: pending
priority: p1
issue_id: "001"
tags: [security, authentication]
dependencies: []
---

git工作树

git

您正在开发一个功能分支，但您需要在不丢失当前工作的情况下查看 PR。 Git 工作树让您可以在不同的目录中同时检出多个分支。这项技能可以管理它们（创建、切换、清理），因此您可以进行上下文切换，而无需隐藏或提交半成品代码。

命令

# Create new worktree
bash scripts/worktree-manager.sh create feature-login

# List worktrees
bash scripts/worktree-manager.sh list

# Switch to worktree
bash scripts/worktree-manager.sh switch feature-login

# Clean up completed worktrees
bash scripts/worktree-manager.sh cleanup

一体化

  • 与 /review 用于单独的 PR 分析
  • 与 /work 用于并行功能开发

要求

  • Git 2.8+（用于工作树支持）
  • 工作树存储在 .worktrees/ 目录
skill: git-worktree

图像生成 (1)

使用 AI 生成图像。不是您在 Unsplash 上找到的库存照片 - 您描述和模型创建的图像。

Gemini-imagegen

人工智能图像

需要带有特定文字的徽标吗？大理石表面上的产品模型？卡哇伊风格的插图？该技能封装了 Google 的 Gemini 图像生成 API。你描述你想要什么，它就会生成它。您可以编辑现有图像、优化多轮图像或根据参考图像进行合成。全部通过简单的 Python 脚本完成。

特征

文本到图像的生成
图像编辑和操作
多轮迭代细化
多个参考图像（最多 14 个）
Google 搜索接地（专业版）

可用型号

模型 解决 最适合
gemini-2.5-flash-image 1024px 速度快、大批量的任务
gemini-3-pro-image-preview 高达 4K 专业资产，复杂指令

快速入门

# Text-to-image
python scripts/generate_image.py "A cat wearing a wizard hat" output.png

# Edit existing image
python scripts/edit_image.py input.png "Add a rainbow in the background" output.png

# Multi-turn chat
python scripts/multi_turn_chat.py

图像配置

from google.genai import types

response = client.models.generate_content(
    model="gemini-3-pro-image-preview",
    contents=[prompt],
    config=types.GenerateContentConfig(
        response_modalities=['TEXT', 'IMAGE'],
        image_config=types.ImageConfig(
            aspect_ratio="16:9",  # 1:1, 2:3, 3:2, 4:3, 16:9, 21:9
            image_size="2K"       # 1K, 2K, 4K (Pro only)
        ),
    )
)

提示最佳实践

  • 逼真 - 包括相机详细信息：镜头类型、灯光、角度、情绪
  • 风格化艺术 - 明确指定风格：卡哇伊、卡通阴影、粗体轮廓
  • 图像中的文字 - 明确字体样式和位置（使用 Pro 模型）
  • 产品模型 - 描述照明设置和表面

要求

GEMINI_API_KEY 所需的环境变量
google-genai Python包
pillow 用于图像处理的Python包

所有生成的图像都包含 SynthID 水印。仅图片模式不适用于 Google 搜索接地。

skill: gemini-imagegen
