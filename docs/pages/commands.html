命令参考

这是关于斜杠命令的事情：它们是您需要花费 20 分钟手动完成的工作流程，并被压缩为一行。类型 /plan 一边喝咖啡，一边观看三个代理同时启动来研究您的代码库。这就是重点——自动化实际上可以节省时间，而不是伪装成生产力的忙碌工作。

工作流命令（四）

这就是四大要点：规划你的功能、审查你的代码、完成实施以及整理你所学到的知识。每个专业开发人员都会执行此循环 - 这些命令只会让您更快地完成它。

/plan

您有一个功能请求和一个空白页。该命令将“我们需要 OAuth”转变为一个结构化计划，该计划实际上告诉您要构建什么 - 研究、审查并准备好执行。

论据

[feature description, bug report, or improvement idea]

工作流程

  1. 存储库研究（并行） - 同时启动三个代理：
    • repo-research-analyst - 项目模式
    • best-practices-researcher - 行业标准
    • framework-docs-researcher - 框架文档
  2. SpecFlow 分析 - 跑步 spec-flow-analyzer 对于用户流量
  3. 选择详细程度:
    • 极简 - 简单的错误/小改进
    • 更多的 - 标准功能
    • 很多 - 具有阶段性的主要特征
  4. 写计划 - 另存为 plans/<issue_title>.md
  5. 审查 - 称呼 /plan_review 用于多代理反馈

该命令不编写代码。它只研究并制定计划。

/plan Add OAuth integration for third-party auth
/plan Fix N+1 query in user dashboard

获得一堆代码审查结果或安全审核结果？这会将它们变成可操作的待办事项 - 一次一个，由您决定：创建待办事项、跳过它或修改并重新呈现。

论据

[findings list or source type]

用户决定

  • “是的” - 创建/更新待办事项文件，将状态更改为就绪
  • “下一个” - 从待办事项中跳过和删除
