变更日志

复合工程插件的所有显着变化。该项目如下 语义版本控制 和 保留变更日志 惯例。

v2.6.0

2024-11-26

已删除

  • feedback-codifier 代理人 - 从工作流代理中删除。 代理人数从 24 人减少至 23 人。

v2.5.0

2024-11-25

额外

  • /report-bug 命令 - 用于报告错误的新斜杠命令 复合工程插件。提供收集错误信息的结构化工作流程 通过引导式问题，自动收集环境详细信息，并创建GitHub EveryInc/every-marketplace 存储库中的问题。

v2.4.1

2024-11-24

改进

  • 设计迭代代理 - 添加了重点截图指导：始终捕获 仅目标元素/区域而不是整页屏幕截图。包括 browser_resize 建议、使用 browser_snapshot refs 的针对元素的屏幕截图工作流程，以及 明确指示永远不要使用全页模式。

v2.4.0

2024-11-24

固定的

  • MCP配置 - 将 MCP 服务器移回 plugin.json 以下是来自人类学/生命科学插件的工作示例。
  • 上下文7 URL - 更新为使用具有正确端点 URL 的 HTTP 类型。

2024-11-24

改变了

v2.0.0

2024-11-24 主要版本

重大重组，将多个来源的代理、指挥和技能整合为 一个单一的、组织良好的插件。

额外

新代理（七）

  • design-iterator - 通过系统设计迭代，迭代细化UI组件
  • design-implementation-reviewer - 验证 UI 实现是否符合 Figma 设计规范
  • figma-design-sync - 将 Web 实现与 Figma 设计同步
  • bug-reproduction-validator - 系统地重现和验证错误报告
  • spec-flow-analyzer - 分析用户流量并确定规范中的差距
  • lint - 对 Ruby 和 ERB 文件运行 linting 和代码质量检查
  • ankane-readme-writer - 按照 Ankane 风格的 Ruby gems 模板创建自述文件

新命令（九）

  • /changelog - 为最近的合并创建引人入胜的变更日志
  • /plan_review - 并行多代理计划审查
  • /resolve_parallel - 并行解决TODO注释
  • /resolve_pr_parallel - 并行解决 PR​​ 评论
  • /reproduce-bug - 使用日志和控制台重现错误
  • /prime - 启动/设置命令
  • /create-agent-skill - 创建或编辑克劳德代码技能
  • /heal-skill - 修复技能文档问题
  • /codify - 记录知识库已解决的问题

新技能 (10)

  • andrew-kane-gem-writer - 按照 Andrew Kane 的模式编写 Ruby gems
  • codify-docs - 将已解决的问题捕获为分类文档
  • create-agent-skills - 创建克劳德代码技能的专家指导
  • dhh-ruby-style - 以 DHH 的 37signals 风格编写 Ruby/Rails 代码
  • dspy-ruby - 使用 DSPy.rb 构建类型安全的 LLM 应用程序
  • every-style-editor - 审查副本以确保每个人的风格指南合规性
  • file-todos - 基于文件的待办事项跟踪系统
  • frontend-design - 创建生产级前端界面
  • git-worktree - 管理 Git 工作树以进行并行开发
  • skill-creator - 创建有效的克劳德代码技能的指南

改变了

代理商按类别重组

  • review/ （10 名代理）- 代码质量、安全性、性能审核员
  • research/ （四个代理）- 文档、模式、历史分析
  • design/ （三个代理）- UI/设计审查和迭代
  • workflow/ （六名代理）- PR 解决、错误验证、linting
  • docs/ （一名代理）- 自述文件生成
