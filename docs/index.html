常见问题解答

什么是复合工程？

这与大多数团队的工作方式相反。通常情况下，你修复了一个错误，发布它，然后就忘记了。下个月有人遇到了同样的错误并从头开始重新解决它。复合工程意味着每次修复都会教会系统。您的第三次代码审查比第一次更快，因为代理学习了模式。您的第十次安全审核发现了您在第二次审核中遗漏的问题。工作会积累而不是蒸发。

代理与技能有何不同？

代理商 是从事特定工作的同事。安全哨兵进行安全审查。 kieran-rails-reviewer 强制执行 Rails 约定。你直接打电话给他们： claude agent security-sentinel.

技能 这些是克劳德在需要时可以利用的专业知识。 dhh-ruby 风格知道 37 种信号模式。 gemini-imagegen 知道如何生成图像。 Claude 在相关时自动调用它们，或者您可以显式调用它们： skill: dhh-ruby-style.

为什么 MCP 服务器不自动加载？

是的，我们知道。这是当前的限制。解决方法很简单：手动将 MCP 服务器添加到您的 .claude/settings.json 文件。检查自述文件以获取复制粘贴配置。需要 30 秒，你就完成了。

我可以在 Ruby/Rails 以外的语言中使用它吗？

绝对地。除了 Rails 审阅者之外，我们还有 Python 和 TypeScript 审阅者。工作流程命令、研究代理和 Gemini-imagegen 之类的技能并不关心您编写的语言。安全哨兵可以发现 SQL 注入，无论是在 Rails、Django 还是 Laravel 中。

我如何创建自己的代理或技能？

跑步 /create-agent-skill 或调用 create-agent-skills 技能。两者都为您提供模板、实施最佳实践并引导您完成结构。您将在几分钟内拥有一个可用的代理或技能，而不是根据示例进行逆向工程。
